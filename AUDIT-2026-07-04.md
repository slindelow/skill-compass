## 4. The real "autonomous routing" mechanism (why this works without you remembering anything)

Claude auto-invokes skills by matching your request against skill **descriptions** — that part already works. What fails in practice: (a) when several skills match, Claude picks inconsistently; (b) low-signal requests match nothing so no skill fires. The fix is not more model intelligence, it's two mechanical pieces:

1. **A router skill** (`skill-compass`) whose description matches the *failure moments* ("which tool should I use", start of any multi-step task, overlapping candidates) and whose body is the decision table above.
2. **A SessionStart hook** that injects the compact routing table into every session's context, so the priority rules exist *before* any matching happens. This is the same mechanism superpowers uses to enforce itself, and it's the only genuinely autonomous option the harness offers.

Both pieces are in this repo. The skill alone works today; the hook activates when this becomes an installed plugin.
